# MVP Core - Design

## Context

SpiceLab 是一个全新的私密情侣互动游戏平台。当前项目处于从零开始阶段，已有完整的产品文档、技术架构设计和 UI 规范。

**当前状态：**
- 产品规划文档完整（用户画像、市场分析、变现模式）
- 游戏规则已定义（真心话大冒险、飞行棋）
- 技术选型确定（Spring Boot + React/Vue + MySQL）
- UI 规范已定义

**约束：**
- 1 人团队，需要控制复杂度
- MVP 阶段，优先验证核心功能
- 需要快速迭代

## Goals / Non-Goals

**Goals:**
- 实现房间系统：创建、加入、实时同步
- 实现真心话大冒险：轮流、掷骰子、选择、展示题目
- 实现飞行棋：完整规则（起飞、移动、撞子、跳跃、飞行、终点）
- 实现基础题库：30-50 题，支持分级
- 网页版 MVP，移动端友好

**Non-Goals:**
- 用户注册/登录系统（MVP 不需要）
- 付费系统（后续版本）
- 附近的人功能（不做）
- AI 伴侣功能（后续版本）
- APP / 小程序（先做网页版）
- 视频/语音功能（后续版本）
- 云同步功能（MVP 不需要）

## Decisions

### 架构决策

**前后端分离**
- 前端：React + Vite + Tailwind CSS
- 后端：Spring Boot 3.x + WebSocket + MySQL
- 通信：REST API + WebSocket

**状态同步方案**
- 使用 WebSocket 实现房间内实时同步
- 服务端维护房间状态，客户端同步显示
- 断线重连后自动恢复状态

**房间管理方案**
- 房间码：6 位字母数字（如 ABC123）
- 无需登录，通过房间码 + 昵称加入
- 房间过期：24 小时无活动自动清理

### 技术决策

**前端状态管理**
- 使用 Zustand 管理全局状态
- WebSocket 连接状态
- 游戏状态（当前玩家、阶段、数据）

**后端游戏引擎**
- 游戏引擎独立于框架（方便后续开源）
- 使用接口抽象题库来源（方便后续扩展付费题库）
- 状态机管理游戏流程

**数据库设计**
- rooms: 房间信息
- players: 玩家信息
- questions: 题库
- game_records: 游戏记录（可选）

## Risks / Trade-offs

### 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| WebSocket 连接不稳定 | 游戏中断 | 实现断线重连，状态恢复 |
| 1 人团队开发周期长 | 市场窗口错过 | 砍功能，聚焦核心 |
| 飞行棋规则复杂 | 开发时间长 | 分阶段实现，先核心规则 |
| 并发问题 | 状态不一致 | 房间级别加锁，操作序列化 |

### Trade-offs

| 选择 | 放弃 | 原因 |
|------|------|------|
| 无登录系统 | 用户追踪 | MVP 阶段减少摩擦 |
| 房间 24 小时过期 | 长期保存 | 简化存储，降低成本 |
| 前后端分离 | SSR | 前后端独立开发，更灵活 |
| MySQL | PostgreSQL | 熟悉度优先 |
